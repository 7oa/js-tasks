• Task1. HTTP request promisification
    Промифицировать XMLHttpRequest. Сделать два запроса таким образом, чтобы получить ответ одновременно от обоих.
  • Task2. Input fields with history of changes
    Дана HTML форма, состоящая из нескольких input полей. Каждый из input полей должен хранить историю своих изменений (`onchange` event) в замыкании. Пользователь должен иметь возможность передвигаться по истории с помощью кнопок "вперед" и "назад". Также необходимо учесть неактивное состояние кнопок навигации по истории, когда невозможно выполнить действие перемещения вперед или назад.
  • Task3. Retry operation
    Создать хелпер-функцию `executeFn()`, которая вызывает переданную в аргументах асинхронную функцию `fn()` до тех пор пока она выполнится без ошибок. Максимальное количество попыток вызова ограничено и также передается аргументом в функцию `executeFn()`. В случае если все попытки закончились неудачей, то необходимо пробросить ошибку полученную от вызываемой функции.
  • Task4. Password generator
    Необходимо создать небольшое приложение, которое позволяет пользователю сгенерировать сложный и безопасный пароль. Приложение состоит из формы с одним input полем для ввода значения длины пароля и кнопки "Submit". Сгенерированный пароль должен отображаться в input поле с кнопкой "Copy to buffer" для копирования пароля в буфер обмена.
    Безопасным пароль считается в том случае, если:
    - содержит хотя бы один символ в нижнем регистре: [abcdefghijklmnopqrstuvwxyz]
    - содержит хотя бы один символ в верхнем регистре: [ABCDEFGHIJKLMNOPQRSTUVWXYZ]
    - содержит хотя бы одну цифру: [0123456789]
    - содержит хотя бы один специальный символ: [ !#$%&()*+,-./:;<=>?@[\]^_{|}~]
    - является уникальным по отношению к сгенерированным в этой же сессии
    - содержит не менее 4 символов
  • Task5. Users list
    Страница управления пользователями в приложении состоит из таблицы, содержащей список пользователей с информацией о имени, email, телефоне и статусе аккаунта. При клике на ячейку, содержащую текстовые данные, в таблице должен появляться prompt, с помощью которого можно изменить текст выбранной ячейки. Также любого пользователя из списка можно удалить или поменять его статус, то есть активировать или заблокировать аккаунт. Для добавления новых пользователей используется форма под таблицей. Задачу требуется решить с помощью делегирования событий.
